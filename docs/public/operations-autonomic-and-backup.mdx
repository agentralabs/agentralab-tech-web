<Callout type="info" title="Why this page matters">
Install is step one. Reliability comes from artifact protection, configuration backup, and predictable operational defaults.
</Callout>

## What you should protect

- `~/.brain.amem` (default AgenticMemory brain file)
- MCP client configs (for example `~/.claude/mcp.json` when present)
- Runtime artifacts you generate (`.avis`, `.acb`) in your workspace paths
- Runtime logs/health snapshots when enabled

## Fast backup workflow (macOS/Linux)

<Steps>
  <Step number={1} title="Create backup folders">
    <Command
      title="Prepare backup directories"
      description="Keeps runtime and config backups separated by purpose."
      code={`mkdir -p "$HOME/.agentra-backups/memory"
mkdir -p "$HOME/.agentra-backups/mcp"
mkdir -p "$HOME/.agentra-backups/artifacts"`}
    />
  </Step>

  <Step number={2} title="Backup the memory brain file">
    <Command
      title="Backup ~/.brain.amem"
      description="Timestamped backup file for safe rollback and migration."
      code={`cp "$HOME/.brain.amem" "$HOME/.agentra-backups/memory/brain-$(date +%F-%H%M%S).amem"`}
    />
  </Step>

  <Step number={3} title="Backup MCP config">
    <Command
      title="Backup Claude MCP config (if present)"
      description="Use the same pattern for other MCP client config files you discover."
      code={`if [ -f "$HOME/.claude/mcp.json" ]; then
  cp "$HOME/.claude/mcp.json" "$HOME/.agentra-backups/mcp/claude-mcp-$(date +%F-%H%M%S).json"
fi`}
    />
  </Step>

  <Step number={4} title="Backup workspace artifacts">
    <Command
      title="Backup .avis and .acb outputs"
      description="Adjust path to where your project writes runtime artifacts."
      code={`find "$PWD" -type f \( -name "*.avis" -o -name "*.acb" \) -maxdepth 4 -print0 | \
  xargs -0 -I {} cp "{}" "$HOME/.agentra-backups/artifacts/"`}
    />
  </Step>
</Steps>

## Restore workflow

<Command
  title="Restore memory brain"
  description="Stop active clients first, then restore the selected backup."
  code={`cp "$HOME/.agentra-backups/memory/<backup-file>.amem" "$HOME/.brain.amem"`}
/>

<Command
  title="Restore MCP config"
  description="Restore your chosen MCP config backup and restart the desktop client."
  code={`cp "$HOME/.agentra-backups/mcp/<backup-file>.json" "$HOME/.claude/mcp.json"`}
/>

<Callout type="success" title="Verification after restore">
Run a binary check and one MCP tool call to confirm runtime state is healthy after restore.
</Callout>

## Discover MCP files after install

If you are unsure where client MCP config lives, scan common locations:

<Command
  title="Find likely MCP config files"
  description="Searches common home/config paths for MCP and desktop config names."
  code={`find "$HOME" -maxdepth 5 \( -name "*mcp*.json" -o -name "claude_desktop_config.json" -o -name "settings.json" \) 2>/dev/null | head -n 80`}
/>

<Command
  title="Find Agentra MCP entries in configs"
  description="Quickly identify files already containing Agentra runtime entries."
  code={`rg -n "agentic-memory|agentic-vision|acb-mcp|agentra" "$HOME/.claude" "$HOME/.config" "$HOME/Library/Application Support" 2>/dev/null`}
/>

## Autonomic and backup controls (Codebase runtime)

These environment variables are available for codebase operational posture:

- `ACB_AUTONOMIC_PROFILE` (`desktop`, `cloud`, `aggressive`)
- `ACB_STORAGE_MIGRATION_POLICY` (`auto-safe`, `strict`, `off`)
- `ACB_AUTO_BACKUP`
- `ACB_AUTO_BACKUP_RETENTION`
- `ACB_AUTO_BACKUP_DIR`
- `ACB_COLLECTIVE_CACHE_MAINTENANCE_SECS`
- `ACB_HEALTH_LEDGER_DIR` (or `AGENTRA_HEALTH_LEDGER_DIR`)
- `ACB_HEALTH_LEDGER_EMIT_SECS`
- `ACB_SLA_MAX_REGISTRY_OPS_PER_MIN`

Default health ledger location:

- `~/.agentra/health-ledger`

<Callout type="warning" title="Operational guardrail">
Treat backup directories as sensitive: they may contain memory context, runtime artifacts, and configuration details. Use encrypted storage for long-term retention.
</Callout>
