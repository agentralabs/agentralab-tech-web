<Callout type="info" title="Primary recommendation">
Use hosted installers first. They keep install behavior consistent across all public docs, README instructions, and web quickstart surfaces.
</Callout>

## Hosted installers

<Command
  title="Install all three runtimes"
  description="Run each command independently. This gives users the option to keep projects standalone."
  code={`curl -fsSL https://agentralabs.tech/install/memory | bash
curl -fsSL https://agentralabs.tech/install/vision | bash
curl -fsSL https://agentralabs.tech/install/codebase | bash`}
/>

## Fallback behavior

If a platform-specific release artifact is unavailable, installers fall back to source-based install (`cargo install` from canonical repo source).

<CliOutput
  title="Fallback example"
  output={`[info] No GitHub release found; using source fallback.
[info] Installing from source...
[ok] Build complete.
[ok] Binary installed in ~/.local/bin`}
/>

## Direct package channels

### Rust crates

<Command
  title="Rust channel"
  description="Direct installation from crates.io for environments that manage binaries via cargo."
  code={`cargo install agentic-memory agentic-memory-mcp
cargo install agentic-vision-mcp
cargo install agentic-codebase`}
/>

### Python helper (memory installer)

<Command
  title="Python helper channel"
  description="Convenient memory setup helper for teams standardizing on Python-based agent tooling."
  code={`pip install amem-installer
amem-install install --auto`}
/>

## Post-install expectations

- platform detection summary
- binary destination path
- MCP merge status
- restart guidance
- optional runtime validation command

<Callout type="warning" title="Guardrail">
If command output diverges from these expectations, capture full terminal output and report it via <a href="/docs/feedback">Feedback and Community</a>.
</Callout>
