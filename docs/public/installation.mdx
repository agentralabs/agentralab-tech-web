## Fallback behavior

If a platform-specific release artifact is unavailable, installers fall back to source-based install (`cargo install` from canonical repo source).

<CliOutput
  title="Fallback example"
  output={`[info] No GitHub release found; using source fallback.
[info] Installing from source...
[ok] Build complete.
[ok] Binary installed in ~/.local/bin`}
/>

## Post-install expectations

- platform detection summary
- binary destination path
- MCP merge status
- restart guidance
- optional runtime validation command

<Callout type="warning" title="Guardrail">
If command output diverges from these expectations, capture full terminal output and report it via <a href="/docs/feedback">Feedback and Community</a>.
</Callout>
