Common issues and solutions for AgenticComm.

## Installation Issues

### Binary not found after install

Ensure `~/.local/bin` is in your PATH:

```bash
export PATH="$HOME/.local/bin:$PATH"
# Add to ~/.bashrc or ~/.zshrc for persistence
```

### Install script fails with "jq not found"

The installer needs `jq` or `python3` for MCP config merging:

```bash
# macOS
brew install jq

# Ubuntu/Debian
sudo apt install jq

# Or use python3 (usually pre-installed)
python3 --version
```

### Cargo build fails

Ensure you have the latest stable Rust toolchain:

```bash
rustup update stable
```

## MCP Server Issues

### Server not appearing in MCP client

1. Verify the binary exists: `ls ~/.local/bin/agentic-comm-mcp`
2. Check config was merged: look for `agentic-comm` in your MCP client config
3. Restart your MCP client completely (not just reload)
4. Run manually to check for errors: `agentic-comm-mcp serve`

### "AGENTIC_TOKEN required" error

This occurs in server profile mode. Set the token:

```bash
export AGENTIC_TOKEN="$(openssl rand -hex 32)"
```

### Server crashes on startup

Check for stale lock files:

```bash
ls -la ~/.comms.acomm.lock
# Remove stale locks (check PID first)
rm ~/.comms.acomm.lock
```

## File Format Issues

### "Invalid magic bytes" error

The file is not a valid `.acomm` file. Check:

```bash
xxd -l 4 agents.acomm
# Should show: 4143 4f4d (ACOM)
```

### "Version mismatch" error

The file was created with a newer version of AgenticComm. Update your installation:

```bash
curl -fsSL https://agentralabs.tech/install/comm | bash
```

### File appears empty after save

Ensure `save()` is called explicitly in Python/FFI usage. The MCP server auto-saves.

## Communication Issues

### Messages not appearing in channel

Check that the channel exists and the sender is using the correct channel name:

```bash
acomm channel list
acomm receive "channel-name" --limit 5
```

### Search returning no results

Full-text search is case-insensitive but requires at least one matching term:

```bash
# Try broader search terms
acomm search "deploy"

# Check message count
acomm stats
```

### Broadcast not reaching all channels

Broadcast sends to all channels that exist at the time of the call. Channels created after the broadcast do not receive the message.

### Subscription not working

Verify the subscription exists:

```bash
acomm subscribe list "channel-name"
```

## Performance Issues

### Slow search with large stores

Search scales linearly with message count. For stores with more than 10K messages:

1. Limit search to specific channels: `acomm search "query" --channel "dev-tasks"`
2. Archive old messages: `acomm export --channel "old-channel" > archive.json`
3. Run compaction: `acomm compact`

### High disk usage

Run compaction to reclaim space from deleted channels and messages:

```bash
acomm compact
```
