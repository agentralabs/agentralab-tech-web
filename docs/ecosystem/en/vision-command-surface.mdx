Install commands are documented in [Installation](/docs/en/vision-installation).

## Binary

- `agentic-vision-mcp`

## Top-level commands

```bash
agentic-vision-mcp serve
agentic-vision-mcp validate
agentic-vision-mcp info
agentic-vision-mcp completions
agentic-vision-mcp repl
```

## Common options

- `-v, --vision <file.avis>`
- `--model <clip_model.onnx>`
- `--log-level trace|debug|info|warn|error`

## Runtime budget controls

```bash
export CORTEX_STORAGE_BUDGET_MODE=auto-rollup
export CORTEX_STORAGE_BUDGET_BYTES=2147483648
export CORTEX_STORAGE_BUDGET_HORIZON_YEARS=20
export CORTEX_STORAGE_BUDGET_TARGET_FRACTION=0.85
```

## Key tools exposed by MCP

- `vision_capture` (now returns `quality_score`)
- `vision_query` (supports `description_contains`, `min_quality`, `sort_by`)
- `vision_health` (quality + staleness + linkage summary)
- `vision_similar`
- `vision_compare`
- `vision_diff`
- `vision_ocr`
- `vision_link`
- `session_start`
- `session_end`

## Grounding Tools (v0.2)

| Tool | Purpose |
|------|---------|
| `vision_ground` | Verify claim has visual evidence |
| `vision_evidence` | Get evidence for a query |
| `vision_suggest` | Find similar captures (typo correction) |

## Workspace Tools (v0.2)

| Tool | Purpose |
|------|---------|
| `vision_workspace_create` | Create multi-vision workspace |
| `vision_workspace_add` | Add `.avis` file to workspace |
| `vision_workspace_list` | List loaded vision contexts |
| `vision_workspace_query` | Query across all contexts |
| `vision_workspace_compare` | Compare item between contexts |
| `vision_workspace_xref` | Find capture distribution across contexts |

## Context-Capture Tool

| Tool | Purpose |
|------|---------|
| `observation_log` | Record why a visual analysis is being performed |

{
