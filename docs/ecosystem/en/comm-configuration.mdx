AgenticComm configuration options for all runtime modes.

## Environment Variables

| Variable | Default | Allowed Values | Effect |
|----------|---------|----------------|--------|
| `ACOMM_FILE` | `~/.comms.acomm` | Path to `.acomm` file | Explicit store file path |
| `ACOMM_MAX_CHANNELS` | `10000` | Positive integer | Maximum channels per store |
| `ACOMM_MAX_MESSAGE_SIZE` | `1048576` (1 MB) | Positive integer (bytes) | Maximum message content size |
| `ACOMM_COMPRESSION` | `lz4` | `lz4`, `none` | Message compression algorithm |
| `RUST_LOG` | `info` | `trace`, `debug`, `info`, `warn`, `error` | Logging verbosity |
| `AGENTRA_WORKSPACE_ROOT` | Auto-detected | Directory path | Override workspace root detection |
| `AGENTRA_PROJECT_ROOT` | Auto-detected | Directory path | Override project root detection |
| `AGENTIC_TOKEN` | None | String | Auth token for server profile |
| `AGENTIC_TOKEN_FILE` | None | File path | Auth token file for server profile |

## MCP Server Configuration

The MCP server (`agentic-comm-mcp`) accepts the following arguments:

```json
{
  "mcpServers": {
    "agentic-comm": {
      "command": "agentic-comm-mcp",
      "args": ["serve"]
    }
  }
}
```

## File Location Resolution

AgenticComm resolves the `.acomm` file in this order:

1. `ACOMM_FILE` environment variable (if set)
2. `--store` CLI flag (if provided)
3. `~/.comms.acomm` (default)

## Install Profiles

| Profile | CLI | MCP Server | Claude Desktop | Claude Code | Token Auth |
|---------|-----|------------|----------------|-------------|------------|
| **desktop** | Yes | Yes | Configured | Configured | No |
| **terminal** | Yes | Yes | No | Configured | No |
| **server** | Yes | Yes | No | No | Required |

## Runtime Modes

| Mode | Trigger | Behavior |
|------|---------|----------|
| `desktop` | Default | Full MCP server, auto-session |
| `terminal` | `--mode terminal` | CLI-optimized, no auto-session |
| `server` | `AGENTIC_TOKEN` set | Token-gated, no desktop config |

## Compression

Message content is LZ4-compressed by default. Disable with `ACOMM_COMPRESSION=none` for debugging or when content is already compressed.

## Concurrent Access

The store uses file locking for safe concurrent access. Multiple MCP clients can share the same `.acomm` file. Stale locks (from crashed processes) are automatically detected and recovered via PID validation.
