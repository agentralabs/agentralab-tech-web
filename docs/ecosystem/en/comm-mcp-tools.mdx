AgenticComm exposes 17 tools through the MCP protocol via `agentic-comm-mcp`.

## Channel Tools

### `comm_channel_create`

Create a new communication channel.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Unique channel name |
| `description` | string | No | Human-readable description |

**Returns:** `{ "name": "dev-tasks", "description": "Development task queue" }`

### `comm_channel_list`

List all channels in the store.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `limit` | number | No | Maximum results (default: 50) |

### `comm_channel_delete`

Delete a channel and all its messages.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Channel name to delete |

### `comm_channel_info`

Get detailed information about a channel.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Channel name |

## Message Tools

### `comm_send`

Send a message to a channel.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | Yes | Target channel name |
| `content` | string | Yes | Message content |
| `sender` | string | No | Sender identifier (default: auto-detected) |

**Returns:** `{ "id": 42, "channel": "dev-tasks", "timestamp": "2026-02-28T10:00:00Z" }`

### `comm_receive`

Receive messages from a channel.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | Yes | Channel name |
| `limit` | number | No | Maximum messages to receive (default: 10) |

### `comm_peek`

Read messages without consuming them.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | Yes | Channel name |
| `limit` | number | No | Maximum messages (default: 10) |

### `comm_broadcast`

Send a message to all channels simultaneously.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `content` | string | Yes | Message content to broadcast |
| `sender` | string | No | Sender identifier |

## Subscription Tools

### `comm_subscribe`

Subscribe an agent to a channel.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | Yes | Channel to subscribe to |
| `agent` | string | Yes | Agent identifier |

### `comm_unsubscribe`

Unsubscribe an agent from a channel.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | Yes | Channel to unsubscribe from |
| `agent` | string | Yes | Agent identifier |

### `comm_subscription_list`

List subscriptions for a channel or agent.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | No | Filter by channel |
| `agent` | string | No | Filter by agent |

## Search Tools

### `comm_search`

Full-text search across messages.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | Search query text |
| `channel` | string | No | Limit search to specific channel |
| `limit` | number | No | Maximum results (default: 20) |

## Administration Tools

### `comm_store_info`

Get store statistics and metadata.

**Returns:** `{ "channels": 8, "messages": 142, "subscriptions": 5, "file_size": 12800 }`

### `comm_store_health`

Run integrity checks on the store.

**Returns:** `{ "status": "healthy", "header_valid": true, "index_valid": true }`

### `comm_export`

Export store data as JSON.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | No | Export specific channel only |
| `format` | string | No | `json` (default) |

### `comm_import`

Import data into the store.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `data` | string | Yes | JSON string of communication data |
| `merge` | boolean | No | Merge with existing (default: true) |

### `comm_log`

Log a communication observation or context note.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `content` | string | Yes | The observation to log |
| `topic` | string | No | Category or topic |
