AgenticComm exposes communication data through the `acomm://` URI scheme in the MCP Resources API.

## URI Scheme

All resources use the `acomm://` prefix.

## Available Resources

### `acomm://channels`

Returns all channels in the current store.

**Format:** JSON array of channel objects.

```json
[
  {
    "name": "dev-tasks",
    "description": "Development task queue",
    "message_count": 42,
    "subscriber_count": 3,
    "created_at": "2026-02-20T10:00:00Z",
    "last_message_at": "2026-02-28T14:30:00Z"
  }
]
```

### `acomm://channels/{name}`

Returns a single channel by name with full details.

### `acomm://channels/{name}/messages`

Returns messages in a specific channel.

```json
[
  {
    "id": 1,
    "channel": "dev-tasks",
    "content": "Build the login page with React",
    "sender": "planner-agent",
    "timestamp": "2026-02-28T10:00:00Z"
  }
]
```

### `acomm://channels/{name}/messages/latest`

Returns the most recent message in a channel.

### `acomm://subscriptions`

Returns all subscriptions across all channels.

```json
[
  {
    "channel": "dev-tasks",
    "agent": "worker-agent",
    "subscribed_at": "2026-02-20T10:05:00Z"
  }
]
```

### `acomm://subscriptions/{agent}`

Returns all channels an agent is subscribed to.

### `acomm://stats`

Returns store statistics.

```json
{
  "channels": 8,
  "messages": 142,
  "subscriptions": 5,
  "file_path": "/Users/user/.comms.acomm",
  "file_size_bytes": 12800
}
```

### `acomm://health`

Returns store health status.

```json
{
  "status": "healthy",
  "header_valid": true,
  "channel_index_valid": true,
  "message_index_valid": true,
  "content_blocks_valid": true
}
```

## Resource Templates

### `acomm://channels/{name}/messages?limit={n}`

Limit the number of messages returned.

### `acomm://channels/{name}/messages?sender={sender}`

Filter messages by sender.

### `acomm://search?q={query}`

Search messages across all channels via resource URI.

## Cross-Sister Resources

When running alongside other Agentra sisters, AgenticComm resources can be referenced:

- Memory nodes can link to `acomm://channels/{name}` for communication context
- Identity receipts can reference `acomm://channels/{name}/messages/{id}` for audit trails
- Time schedules can link to `acomm://channels/{name}` for deadline-aware messaging
