AgenticComm provides 3 built-in MCP prompts that agents can invoke for structured communication workflows.

## `comm_coordinate`

Coordinate a multi-agent task through structured channel communication.

### Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `task` | string | Yes | Description of the task to coordinate |
| `agents` | string | No | Comma-separated list of participating agents |
| `channel` | string | No | Channel to use (created if missing) |

### Behavior

The prompt instructs the agent to:

1. Create or join the specified channel
2. Break the task into delegatable subtasks
3. Send each subtask to the channel with clear expectations
4. Monitor for completion messages from participating agents
5. Aggregate results and report final status

### Example

```json
{
  "name": "comm_coordinate",
  "arguments": {
    "task": "Build and deploy the authentication module",
    "agents": "developer, reviewer, deployer",
    "channel": "auth-pipeline"
  }
}
```

## `comm_review`

Review communication health across all channels.

### Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `scope` | string | No | `all`, `active`, `stale` (default: `all`) |
| `window_hours` | number | No | Look-back window in hours (default: 24) |

### Behavior

The prompt instructs the agent to:

1. List all channels with recent activity
2. Identify stale channels (no messages in window)
3. Count unread messages per channel
4. Report on subscription coverage (agents without subscriptions)
5. Flag channels with high message volume that may need attention
6. Suggest cleanup actions for unused channels

### Example

```json
{
  "name": "comm_review",
  "arguments": {
    "scope": "active",
    "window_hours": 48
  }
}
```

## `comm_broadcast_status`

Broadcast a structured status update to all channels or subscribers.

### Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `status` | string | Yes | Current status or update to broadcast |
| `category` | string | No | `info`, `warning`, `error`, `success` (default: `info`) |
| `channels` | string | No | Comma-separated channel filter (default: all) |

### Behavior

The prompt instructs the agent to:

1. Format the status message with timestamp and category
2. Send to all specified channels (or broadcast to all)
3. Log the broadcast for audit purposes
4. Report delivery confirmation per channel

### Example

```json
{
  "name": "comm_broadcast_status",
  "arguments": {
    "status": "Deployment v1.2.0 complete on staging. Production deploy scheduled for 14:00 UTC.",
    "category": "success"
  }
}
```
