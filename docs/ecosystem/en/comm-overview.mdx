# AgenticComm Overview

AgenticComm solves a fundamental problem in multi-agent systems: **agents cannot communicate with each other across sessions, models, or environments**.

Every AI agent today is an island. When one agent finishes work, the next has no structured way to receive context, tasks, or results. Teams resort to shared files, webhooks, or database queues — none designed for agent coordination.

## What AgenticComm provides

AgenticComm gives every AI agent a persistent, structured communication layer through the `.acomm` binary file format:

- **Named channels** with independent message streams and subscriber lists
- **Typed messages** — commands, queries, responses, notifications, and acknowledgments
- **Subscription management** — agents subscribe to channels and poll for updates
- **Full-text search** across all messages with timestamp filtering
- **Broadcast** to all channels simultaneously
- **Binary format** with LZ4 compression and append-only writes for crash safety

## 17 MCP tools

When connected via MCP, your LLM client gains access to 17 communication tools organized in 5 categories:

| Category | Tools |
|----------|-------|
| **Channel** | `comm_channel_create`, `comm_channel_list`, `comm_channel_delete` |
| **Messaging** | `comm_send`, `comm_receive`, `comm_peek` |
| **Pub/Sub** | `comm_subscribe`, `comm_unsubscribe`, `comm_subscriptions` |
| **Query** | `comm_search`, `comm_history` |
| **Context** | `comm_broadcast`, `comm_poll`, `comm_store_info`, `comm_store_create`, `comm_health`, `conversation_log` |

## Use cases

- **Task delegation**: One agent posts work items, another picks them up
- **Code review handoff**: Developer agent sends PR for reviewer agent
- **Status broadcasting**: System-wide announcements across all channels
- **Multi-agent coordination**: Structured workflows with acknowledgments
- **Audit trail**: Searchable history of all inter-agent communication
