## Server takeover handshake (required for cloud/runtime agents)

For hosted agents (Cloudflare, containers, VM services), local artifact files are not directly readable from your laptop. Use explicit auth + artifact sync:

<Command
  title="Required server runtime variables"
  description="Set these before starting MCP runtimes in server mode."
  code={`export AGENTRA_RUNTIME_MODE=server
export AGENTIC_TOKEN="$(openssl rand -hex 32)"
export AGENTRA_ARTIFACT_DIRS="/srv/agentra:/data/brains"
# optional alternative:
# export AGENTIC_TOKEN_FILE="/etc/agentra/token"`}
/>

<Command
  title="Sync local artifacts to server runtime"
  description="Cloud runtime cannot directly read local .amem/.avis/.acb files; sync first."
  code={`./sync_artifacts.sh --target=<server-path-or-rsync-target>`}
/>

## Fallback behavior

If a platform-specific release artifact is unavailable, installers fall back to source-based install (`cargo install` from canonical repo source).

<CliOutput
  title="Fallback example"
  output={`[info] No GitHub release found; using source fallback.
[info] Installing from source...
[ok] Build complete.
[ok] Binary installed in ~/.local/bin`}
/>

## Post-install expectations

- platform detection summary
- binary destination path
- MCP merge status
- restart guidance
- optional runtime validation command
- when to restart MCP clients after install
- optional feedback path for install quality reports

<Callout type="warning" title="Guardrail">
If command output diverges from these expectations, capture full terminal output and report it via <a href="/docs/feedback">Feedback and Community</a>.
</Callout>
