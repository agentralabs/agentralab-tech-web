## 服务端接管握手（云端 Agent 必须）

对于云端/托管 Agent（Cloudflare、容器、VM），服务端无法直接读取你本地电脑的工件文件。必须先做鉴权并同步工件：

<Command
  title="服务端运行必填变量"
  description="在 server 模式启动 MCP 前设置。"
  code={`export AGENTRA_RUNTIME_MODE=server
export AGENTIC_TOKEN="$(openssl rand -hex 32)"
export AGENTRA_ARTIFACT_DIRS="/srv/agentra:/data/brains"
# 可选文件方式：
# export AGENTIC_TOKEN_FILE="/etc/agentra/token"`}
/>

<Command
  title="同步本地工件到服务端"
  description="云端运行时不能直接读取本地 .amem/.avis/.acb，必须先同步。"
  code={`./sync_artifacts.sh --target=<server-path-or-rsync-target>`}
/>

## 回退行为

当平台对应的发布产物不可用时，安装器会回退到源码安装（从官方仓库执行 `cargo install`）。

<CliOutput
  title="回退示例"
  output={`[info] No GitHub release found; using source fallback.
[info] Installing from source...
[ok] Build complete.
[ok] Binary installed in ~/.local/bin`}
/>

## 安装后应看到的结果

- 平台识别摘要
- 二进制安装路径
- MCP 合并状态
- 重启提示
- 可选运行时验证命令
- 安装后何时需要重启 MCP 客户端
- 可选反馈入口（非强制）

如果输出与预期不一致，请保留完整终端输出并在 <a href="/docs/feedback">反馈与社区</a> 提交。
</Callout>
