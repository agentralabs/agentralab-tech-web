**日期：** 2026 年 2 月 22 日
**范围：** OpenClaw TUI 接管行为、通用 MCP 接线、Agentra 命令 UX 和操作备份/恢复。

## 1) 运行时间接管和会话重新同步

实现了运行时行为，因此 Agentra 接管在会话启动时是自动且可重复的。

- `openclaw tui` 现在自动检测工件和二进制文件中姐妹的存在。
- 检测支持工件：`.acb`、`.amem`、`.avis`。
- 检测支持二进制文件：`acb`、`acb-mcp`、`amem`、`agentic-memory-mcp`、`agentic-vision-mcp`。
- 运行时间重新同步连续运行（定期重新检测），因此当环境发生变化时页脚状态会更新。
- 完全控制状态源自活跃姐妹（不仅是3/3场景）。
- 用户可以随时释放/重新启用控制，而无需卸载任何内容。

## 2) 动态每姐妹禁用/启用覆盖

在运行时添加了非破坏性的每姐妹控制。

- 支持独立禁用任何姐妹：`codebase`、`memory`、`vision`。
- 禁用覆盖优先于工件/二进制检测。
- 由配置文件支持：
- `~/.openclaw/sisters-disable.json`
- 任选环境覆盖：
- `OPENCLAW_SISTERS_DISABLE_FILE`
- `OPENCLAW_SISTERS_DISABLE`（和 `OPENCLAW_DISABLE_SISTERS`）

结果：用户可以保持安装完整，暂时禁用姐妹，然后立即重新启用。

## 3) TUI 页脚和运行时消息传递改进

状态输出现在是明确的并且在操作上有用。

- 页脚显示：
- 连接/断开状态
- 姐妹州 (`On/Off`)
- 检测源标签（`PATH`、`artifact`、`artifact+PATH`、`disabled`、`none`）
- 完全控制计数 (`N/3 enabled`)
- 添加了诊断命令：
- `/agentra where`（支持旧别名）
- 运行时更新消息现在使用 Agentra 措辞：
- `Agentra sisters takeover enabled: ...`
- `Agentra sisters resynced: codebase=..., memory=..., vision=...`
- `Agentra sisters runtime update: codebase=..., memory=..., vision=...`

## 4) Slash Command UX：Agentra 作为主命名空间

将命令 UX 从 `sisters` 命名移至 `agentra` 命名，并具有向后兼容性。

主要命令：

- `/agentra <status|where|on|off|resync|disable|enable>`
- `/agentra-status`
- `/agentra-where`

仍然支持旧别名：

- `/sisters ...`
- `/sisters-status`
- `/sisters-where`

结果：为用户提供更清晰的命名，同时保留现有的工作流程。

## 5) 通用 MCP 接线（不偏袒客户）

跨已发现客户端的标准化 MCP 自动配置行为。

- 客户端检测和合并现在针对桌面和终端环境中的常见 MCP 配置模式。
- Codex MCP 注册/修复流程与基于 JSON 的 MCP 配置集成。
- `agentra doctor --fix` 验证并修复过时/丢失的姐妹 MCP 条目。
- 服务器配置文件仍然是唯一需要明确的身份验证强化指导的模式。

## 6) 新的本机操作命令：备份+恢复

在 `agentra-cli` 中实现了一流的备份操作。

### 添加命令组

- `agentra backup run`
- `agentra backup list`
- `agentra backup verify`
- `agentra backup restore`
- `agentra backup prune`

### 备份行为

`agentra backup run` 在以下位置创建带时间戳的快照：

- 默认根：`~/.agentra-backups`
- snapshot format: `snapshot-<unix-seconds>`

每个快照包含：

- 记忆：
- `~/.brain.amem`（如果存在）
- MCP 配置：
- 发现MCP客户端配置文件
- 运行时工件：
- 工作区扫描中的 `.acb`、`.amem`、`.avis`
- 健康分类档案：
- 来自 `ACB_HEALTH_LEDGER_DIR` / `AGENTRA_HEALTH_LEDGER_DIR` / 默认 `~/.agentra/health-ledger`

每个快照都写：

- `meta/manifest.json`
- `meta/SHA256SUMS.txt`

### 验证行为

`agentra backup verify` 重新散列所有条目并对照清单校验和进行检查。

### 恢复行为

`agentra backup restore <snapshot>` supports selective restore:

- `--memory`
- `--mcp`
- `--artifacts`

安全行为：

- Existing target files are backed up as `.agentra.bak.<timestamp>` before overwrite (unless `--force`).

### 保留策略行为

`agentra backup prune --keep <N>` removes older managed snapshots and keeps newest `N`.

- 任选 `--dry-run` 预览删除。
- 仅修剪托管快照目录 (`snapshot-*`)。

## 7) 安装者安装后 UX 更新

安装完成 (`100% 安装 complete`) 后，用户现在可以获得一致的下一步指导。

- 重新启动MCP客户端/系统
- 验证姐妹服务器是否出现
- 可选的反馈链接

这减少了歧义和首次运行的摩擦。

## 9) 快速命令参考

```bash

/agentra status
/agentra where
/agentra off
/agentra on
/agentra disable memory
/agentra enable memory

agentra backup restore <snapshot> --memory
agentra backup prune --keep 20 --dry-run
agentra backup prune --keep 20
```

## 11) 服务器运行时身份验证+工件同步（已实现）

添加了可执行的服务器运行时检查和对齐的同步指导。

- 新命令：
  - `agentra server preflight [--strict] [--artifact-dir <path>]...`
- 预检验证：
- 启用服务器模式 (`AGENTRA_RUNTIME_MODE=server`)
- 配置身份验证令牌（`AGENTIC_TOKEN` 或令牌文件）
- 配置和现有的工件目录
- 服务器目录中存在的同步工件 (`.acb/.amem/.avis`)
- 可在主机上解析的姊妹 MCP 二进制文件（`acb-mcp`、`agentic-memory-mcp`、`agentic-vision-mcp`）
- 严格模式在失败时退出非零。

还使用下一步信号更新了 `sync_artifacts.sh` 输出：

- 运行`agentra server preflight --strict`
- 重新启动MCP主机/客户端
- 可选的反馈链接
