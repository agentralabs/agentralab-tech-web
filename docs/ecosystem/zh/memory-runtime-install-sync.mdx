此页面定义了安装程序输出、CLI 行为和 Web 文档的预期运行时行为。

## 安装者个人资料

- `desktop`：安装二进制文件并合并检测到的桌面 MCP 配置。
- `terminal`：安装二进制文件，无需特定于桌面的 UX 假设。
- `server`：安装二进制文件，无需写入桌面配置。

## 完成产出合约

安装者必须打印：

1.安装二进制摘要。
2.MCP重启指令。
3. 相关时的服务器身份验证+工件同步指导。
4. 任选反馈指令。

预期完成标记：

```text
Install complete: AgenticMemory (<profile>)
```

## 通用 MCP 配置

```json
{
  "mcpServers": {
    "agentic-memory": {
      "command": "$HOME/.local/bin/agentic-memory-mcp",
      "args": ["serve"]
    }
  }
}
```

## Workspace auto-detection behavior

- 安装er writes `agentic-memory-mcp-agentra` launcher as MCP entrypoint.
- Launcher resolves memory artifact in this order:
1. Explicit override: `AGENTRA_AMEM_PATH` / `AGENTRA_MEMORY_PATH`.
2. Per-workspace default: `<workspace>/.agentra/<workspace-slug>.amem`.
3. Existing local fallbacks (`.brain.amem`, `memory.amem`, home defaults).
- If no file exists yet, launcher routes to per-workspace default path so first run creates and keeps project memory isolated.
- If your MCP client starts outside the project directory, set:

```bash
export AGENTRA_WORKSPACE_ROOT="/absolute/path/to/project"
```

## 服务器验证+同步

```bash
export AGENTIC_TOKEN="$(openssl rand -hex 32)"
```

服务器部署必须在运行前将 `.amem/.acb/.avis` 工件同步到服务器存储。

## 长期存储预算策略

要长期（例如 20 年）以约 1-2 GB 为目标，请配置：

```bash
export AMEM_STORAGE_BUDGET_MODE=auto-rollup
export AMEM_STORAGE_BUDGET_BYTES=2147483648
export AMEM_STORAGE_BUDGET_HORIZON_YEARS=20
export AMEM_STORAGE_BUDGET_TARGET_FRACTION=0.85
```

模式:

- `auto-rollup`：当检测到预算压力时，将已完成的会话自动存档到剧集摘要中。
- `warn`：仅发出警告。
- `off`：禁用策略。

## 提示和反馈捕获政策

要自动将提示/上下文保留到 `.amem` 中，请配置：

```bash
export AMEM_AUTO_CAPTURE_MODE=safe
export AMEM_AUTO_CAPTURE_REDACT=true
export AMEM_AUTO_CAPTURE_MAX_CHARS=2048
```

模式:

- `safe`：以最小的噪音捕获提示模板输入和反馈式字段。
- `full`：捕获更广泛的工具输入上下文（直接 `memory_add` 重复除外）。
- `off`：禁用自动捕获。
