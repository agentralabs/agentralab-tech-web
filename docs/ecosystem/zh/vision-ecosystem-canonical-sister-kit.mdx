状态：规范（规范）
范围：AgenticMemory、AgenticVision、AgenticCodebase 以及每个未来的姐妹存储库

本文件是所有当前和未来姐妹的单一基准合同。新姐妹默认继承此规范，并且在首次公开发布之前必须通过护栏。

## 1. 发布神器合约

- 根据支持的平台发布确定性版本工件。
- Naming format is mandatory: `<project>-<version>-<os>-<arch>.tar.gz`.
- 工件内容必须与自述文件/安装声明完全匹配。
- 如果存储库发布多个二进制文件，则所有必需的二进制文件必须存在于同一版本集中。
- 发布资产必须保持足够稳定以实现安装程序自动化（无临时文件名更改）。

## 2. 安装合同规范

- 需要规范的安装表面：
  - `curl -fsSL https://agentralabs.tech/install/<target> | bash` (backward-compatible default; desktop profile)
  - `curl -fsSL https://agentralabs.tech/install/<target>/desktop | bash`
  - `curl -fsSL https://agentralabs.tech/install/<target>/terminal | bash`
  - `curl -fsSL https://agentralabs.tech/install/<target>/server | bash`
- 安装者必须首先尝试安装发布工件；需要源构建回退。
- MCP 配置行为仅合并，绝不破坏性覆盖。
- 安装者必须打印一致的完成块：
- MCP客户总结
- 通用 MCP 指南（Codex/Cursor/Windsurf/VS Code/Cline/任何 MCP 客户端）
- 快速终端测试命令
- 自述文件安装文档必须包含明确的独立保证，并保留安装程序输出的安装奇偶性。

## 3. 可重复使用的 CI 护栏

- 每个姐妹必须包括：
- `scripts/check-install-commands.sh`
- `scripts/check-canonical-sister.sh`
- `.github/workflows/install-command-guardrails.yml`
- `.github/workflows/canonical-sister-guardrails.yml`
- Guardrails 必须验证自述文件安装命令、安装程序输出要求和规范文档的存在。
- CI 检查必须在 `push` 和 `pull_request` 上运行以获取相关文件。

## 4. 自述文件规范布局

- 截面顺序和顶部结构标准化：
- 英雄图像 -> 徽章 -> 导航 -> 简介 -> 终端窗格
- 剩余的视觉效果必须分布在各个部分；没有堆叠的图像块。
- 必填部分：
  - `安装`
  - `快速开始`
- `How It Works`
- 自述文件安装命令必须与文档和网站快速入门命令匹配。

## 5.MCP 规范简介

- MCP 服务器密钥必须使用稳定的姐妹特定名称。
- 默认命令/参数合同必须在自述文件和安装程序输出中一致记录。
- 安装程序配置合并目标必须保留现有的用户配置。
- 通用 MCP 指导文本必须存在并且与客户无关。

## 6. 包装政策

- 发布渠道（crates/PyPI/npm）必须在每个存储库中显式声明。
- 渠道扩张需要准备门：
- 稳定的公共SDK表面
- 版本控制策略
- 发布自动化和回滚路径
- 支持承诺
- 禁止仅对称发布；渠道必须维护，而不仅仅是开放。

## 7. 版本控制和发布策略

- 语义版本控制是强制性的。
- Git 标签必须遵循 `vX.Y.Z`。
- 发行说明/变更日志必须总结面向用户的变更和安装影响的变更。
- 回滚程序必须存在并经过测试。
- 发布工作流程在发布前必须是绿色的。

## 8.设计资产合约

- 资产风格遵循 Agentra 设计系统：中性基材、单声道系统排版、橙色信号口音。
- 每个存储库所需的基线 SVG 设置：
- 英雄窗格
- 终端面板
- 架构/系统窗格
- 基准/性能面板
- 资产文件名对于自述文件/文档参考应该是稳定且可预测的。

## 9.Env Var 命名空间合约

- 姐妹特定的变量使用姐妹前缀（例如，`AMEM_`、`AVIS_`、`ACB_`）。
- 跨姐妹共享变量使用 `AGENTRA_` 前缀。
- 环境变量文档必须包含一致的表格格式：
- 多变的
- 默认
- 允许值
- 影响

## 10. 新姐妹 Bootstrap

- Canonical bootstrap + README 结构现在位于网络文档中：
- https://agentralabs.tech/docs/ecosystem-feature-reference
- 每个姐妹所需的文档文件夹（最低基线）：
- `docs/quickstart.md`
- `docs/concepts.md`
- `docs/integration-guide.md`
- `docs/faq.md`
- `docs/benchmarks.md`
- `docs/api-reference.md`
- 加上至少一页高级技术页面（`docs/file-format.md` 或 `docs/LIMITATIONS.md`）
- 发布前必须进行网络文档连接：
- 姐妹文档必须可以从 https://agentralabs.tech/docs/sister-docs-catalog 找到
- 同步管道必须包含文档生成中的姐妹 (`docs:sync`) 并发布导航条目
- 首次发布之前的最低引导可交付成果：
- 具有规范布局的自述文件脚手架
- 安装程序脚本
- 安装护栏脚本+工作流程
- 规范姐妹护栏脚本+工作流程
- 上面列出的文档基线
- 新姐妹自述文件必须包括：
- 独立保证副本
- 所有配置文件安装命令（`default`、`desktop`、`terminal`、`server`）
- `agentra` 状态/UI 的可选工作区 UX 注释
- 在所有上述检查通过之前，新姐妹尚未准备好发布。

## 11. Workspace Orchestrator 合约

- 公共协调器行为契约在网络文档中维护：
- https://agentralabs.tech/docs/ecosystem-feature-reference
- 本地回购责任：
- 为每个姐妹保留独立的可安装性
- 保持 Orchestrator UX 可选（绝不是硬性运行时依赖）
- 新姐妹发布门：
- 规范文档文件夹和 Web 文档连接必须在发布之前在 CI 中进行验证。

## 12. Web 文档分组契约

- 公共网络文档必须保持生态系统导航分组且可预测：
  - 总览
- 工作区
- 代理记忆
- 代理代码库
- AgenticVision
  - 反馈与社区
  - 系统架构
  - 用例手册
- 新的姐妹文档必须留在姐妹组下，并且不得重复工作区级别的操作内容。
- 故障排除内容应位于工作区操作文档（矩阵部分）内，而不是作为单独的顶级页面。
- 命令参考应保留在命令界面页面中；安装命令应保留在安装页面中。

## 变更控制

任何例外情况都需要在存储库中明确书面批准以及保留独立安装性和面向用户的命令稳定性的迁移说明。
