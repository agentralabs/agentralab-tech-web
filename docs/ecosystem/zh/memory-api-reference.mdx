`agentic_memory` Python 包 (v0.2.0) 的完整参考。 使用 `pip install agentic-brain` 安装。

＃＃ 脑

用于与 AgenticMemory 图交互的主要类。每个 `Brain` 实例对应于一个 `.amem` 文件。

### 构造函数

```python
Brain(path: str | Path)
```

打开现有的 Brain 文件或在给定路径创建新文件。每次实例化时都会自动启动一个新会话。

**参数：**

|参数|类型 |描述 |
|-----------|------|-------------|
| `path` | `str \| Path` | `.amem` 文件的路径。如果不存在则创建。 |

**引发：** `BrainError` 如果文件存在但已损坏或版本不兼容。

**例子：**

```python
from agentic_memory import Brain

brain = Brain("my_agent.amem")
brain = Brain(Path("/data/agents/assistant.amem"))
```

### add_fact()

```python
Brain.add_fact(
    content: str,
    confidence: float = 1.0,
    metadata: dict[str, str] | None = None
) -> Event
```

存储事实事件——外部观察到或接收到的信息。

**参数：**

|参数|类型 |默认 |描述 |
|-----------|------|---------|-------------|
| `content` | `str` | *必填* |事实的文本内容。 |
| `confidence` | `float` | `1.0` |置信度得分介于 0.0 和 1.0 之间。 |
| `metadata` | `dict[str, str] \| None` | `None` | 任选键值元数据。 |

**返回：** `Event` -- 新创建的事件及其分配的 ID。

＃＃＃ 关联（）

```python
Brain.link(
    source: int,
    target: int,
    edge_type: str | EdgeType,
    weight: float = 1.0
) -> Edge
```

在两个事件之间创建有向的加权边缘。

**参数：**

|参数|类型 |默认 |描述 |
|-----------|------|---------|-------------|
| `source` | `int` | *必填* |源事件的 ID。 |
| `target` | `int` | *必填* |目标事件的ID。 |
| `edge_type` | `str \| EdgeType` | *必填* |以下之一：`"caused_by"`、`"supports"`、`"contradicts"`、`"supersedes"`、`"related_to"`、`"part_of"`、`"temporal_next"`。 |
| `weight` | `float` | `1.0` |边权重介于 0.0 和 1.0 之间。 |

**返回：** `Edge`

**引发：** 如果任一节点 ID 不存在，或者边类型无效，则 `BrainError` 。

＃＃＃ 搜索（）

```python
Brain.search(
    query: str,
    top_k: int = 10,
    event_type: str | EventType | None = None,
    session: int | None = None,
    min_confidence: float = 0.0
) -> list[SearchResult]
```

使用 128 维特征向量对所有事件执行语义相似性搜索。

**参数：**

|参数|类型 |默认 |描述 |
|-----------|------|---------|-------------|
| `query` | `str` | *必填* |自然语言搜索查询。 |
| `top_k` | `int` | `10` |返回的最大结果数。 |
| `event_type` | `str \| EventType \| None` | `None` |将结果过滤为特定事件类型。 |
| `session` | `int \| None` | `None` |将结果过滤到特定会话。 |
| `min_confidence` | `float` | `0.0` |最小置信度阈值。 |

**返回：** `list[SearchResult]` -- 按相似度分数降序排序的结果。

## v0.2 查询扩展方法

v0.2.0 中添加了以下九个方法。所有操作都通过 Rust CLI 后端进行。

### 搜索文本()

```python
Brain.search_text(
    query: str,
    top_k: int = 10,
    event_type: str | None = None
) -> list[TextMatch]
```

BM25 对节点内容的文本搜索。在可用时使用 TermIndex 快速路径（1.58 ms @ 100K 节点），在 v0.1 文件上回退到全扫描慢速路径（122 ms @ 100K）。

**返回：** `list[TextMatch]` -- BM25 分数的结果。

---

### search() （混合）

```python
Brain.search(
    query: str,
    top_k: int = 10
) -> list[HybridMatch]
```

通过倒数排名融合 (RRF) 结合 BM25 和向量相似度的混合搜索。在 100K 节点图表上以 10.83 毫秒测量。

**返回：** `list[HybridMatch]` -- 组合分数、BM25 分数和向量分数的结果。

---

### 中心性()

```python
Brain.centrality(
    metric: str = "pagerank",
    event_type: str | None = None
) -> list[CentralityResult]
```

图中心性分析。支持的指标：`"pagerank"` (34.3 ms @ 100K)、`"degree"` (20.7 ms @ 100K)、`"betweenness"` (10.1 s @ 100K)。

**返回：** `list[CentralityResult]` -- 按中心性得分排名的节点。

---

###最短路径()

```python
Brain.shortest_path(
    src: int,
    dst: int,
    weighted: bool = False
) -> PathResult
```

查找两个节点之间的最短路径。对未加权图使用双向 BFS (104 us @ 100K) 或对加权图使用 Dijkstra (17.6 ms @ 100K)。

**返回：** `PathResult` -- 作为节点 ID 和总成本序列的路径。

---

＃＃＃ 修订（）

```python
Brain.revise(node_id: int) -> RevisionReport
```

反事实信念修正。如果给定节点被撤回，则计算下游级联。只读——无突变。在 100K 节点时测量为 53.4 毫秒。

**返回：** `RevisionReport` -- 受影响的节点、置信度损失、不支持的推论。

---

### 间隙()

```python
Brain.gaps(
    min_severity: float = 0.0,
    max_age_days: int = 30
) -> GapReport
```

确定五类推理弱点：不合理的决策、单一来源的推论、低置信度基础、不稳定的知识和陈旧的证据。

**返回：** `GapReport` -- 按严重性排序的差距以及健康评分。

---

### 类比()

```python
Brain.analogy(
    node_id: int,
    top_k: int = 5
) -> list[Analogy]
```

使用结构指纹（入度、出度、边缘类型分布）结合内容相似性来查找结构相似的过去推理模式。

**返回：** `list[Analogy]` -- 具有相似性分数的前 k 个类似节点。

---

### 合并()

```python
Brain.consolidate(
    dry_run: bool = True
) -> ConsolidationReport
```

图维护：去重、矛盾链接、推理提升、孤儿检测。试运行模式（默认）报告建议的更改而不修改图表。

**返回：** `ConsolidationReport` -- 提议或应用的更改。

---

### 漂移()

```python
Brain.drift(
    topic: str | None = None
) -> DriftReport
```

通过分析取代链和信心轨迹来跟踪信念随时间的演变。在 100K 节点时测量为 68.4 毫秒。

**返回：** `DriftReport` -- 每个主题的稳定性分数、漂移方向、修订频率。

## LLMProvider（抽象基础）

用于实现自定义 LLM 提供商的基类。

```python
from abc import ABC, abstractmethod
from agentic_memory import Event

class LLMProvider(ABC):

    @abstractmethod
    def complete(self, prompt: str, system: str | None = None) -> str:
        """Send a prompt to the LLM and return the completion text.

        Args:
            prompt: The user/input prompt.
            system: Optional system prompt.

        Returns:
            The LLM's response text.

        Raises:
            ProviderError: If the API call fails.
        """
        ...

    @abstractmethod
    def extract_events(self, text: str) -> list[dict]:
        """Extract cognitive events from text.

        The LLM should identify facts, decisions, inferences, etc.
        in the input text and return them as structured dictionaries.

        Args:
            text: The text to extract events from.

        Returns:
            List of dicts with keys: "type", "content", "confidence".

        Raises:
            ProviderError: If extraction fails.
        """
        ...

    def embed(self, text: str) -> list[float] | None:
        """Generate an embedding vector for the given text.

        Optional. If not implemented, the default internal embedding
        model is used. Return a list of 128 floats.

        Args:
            text: The text to embed.

        Returns:
            A 128-dimensional float vector, or None to use the default.
        """
        return None
```

### 实现自定义提供程序

```python
from agentic_memory.integrations import LLMProvider, ProviderError

class MyCustomProvider(LLMProvider):

    def __init__(self, api_url: str, api_key: str):
        self.api_url = api_url
        self.api_key = api_key

    def complete(self, prompt: str, system: str | None = None) -> str:
        # Call your LLM API here
        response = requests.post(
            f"{self.api_url}/completions",
            headers={"Authorization": f"Bearer {self.api_key}"},
            json={"prompt": prompt, "system": system}
        )
        if response.status_code != 200:
            raise ProviderError(f"API error: {response.status_code}")
        return response.json()["text"]

    def extract_events(self, text: str) -> list[dict]:
        extraction_prompt = f"Extract cognitive events from: {text}"
        raw = self.complete(extraction_prompt, system="Extract facts, decisions, inferences...")
        # Parse the LLM output into structured events
        return parse_extraction(raw)

# Usage
provider = MyCustomProvider("https://my-llm.example.com", "my-api-key")
agent = MemoryAgent(brain, provider)
```
