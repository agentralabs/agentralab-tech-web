## 快速安装（一行）

```bash
curl -fsSL https://agentralabs.tech/install/memory | bash
```

下载预构建的 `agentic-memory-mcp` 二进制文件，安装到 `~/.local/bin/`，并将 MCP 服务器配置合并到 Claude Desktop 和 Claude Code 中。内存默认为 `~/.brain.amem`。需要 `curl` 和 `jq`。

### 按环境安装

```bash
# Desktop MCP clients (auto-merge Claude configs)
curl -fsSL https://agentralabs.tech/install/memory/desktop | bash

# Terminal-only (no desktop config writes)
curl -fsSL https://agentralabs.tech/install/memory/terminal | bash

# Remote/server host (no desktop config writes)
curl -fsSL https://agentralabs.tech/install/memory/server | bash
```

### 服务器身份验证和工件同步

云/服务器运行时无法直接从您的笔记本电脑读取文件。

```bash
export AGENTIC_TOKEN="$(openssl rand -hex 32)"
```

All MCP clients must send `Authorization: Bearer <same-token>`.
如果 `.amem/.acb/.avis` 工件是在其他地方创建的，请首先将它们同步到服务器。

## 1. Python SDK（推荐给大多数用户）

Python SDK 为您提供 `Brain` 类和 LLM 集成。需要 **Python 3.10+**。

```bash
pip install agentic-brain
```

### 与 LLM 提供商集成

```bash
pip install agentic-brain[anthropic]   # Claude
pip install agentic-brain[openai]      # GPT
pip install agentic-brain[ollama]      # Local models
pip install agentic-brain[all]         # All providers
```

＃＃＃ 核实

```python
from agentic_memory import Brain

brain = Brain("test.amem")
brain.add_fact("Installation successful", session=1)
print(brain.facts())
```

> **注意：** Python SDK 需要 `amem` 二进制文件（Rust 核心引擎）。 通过下面的步骤 2 安装它，或通过步骤 3 从源代码构建。

## 2.Rust CLI

`amem` 二进制文件是核心引擎。独立使用它或作为 Python SDK 的后端。需要 **Rust 1.70+**。

```bash
cargo install agentic-memory-cli
```

这将安装 `amem` 命令行工具。

＃＃＃ 核实

```bash
amem --help
amem init test.amem
amem add test.amem fact "Installation successful" --session 1
amem info test.amem
```

### 可用命令

**核心命令：**

|命令|描述 |
|:---|:---|
| `amem init` |创建一个新的空 `.amem` 文件 |
| `amem add` |添加认知事件（事实、决定、推理、纠正、技能、情节）|
| `amem link` |在两个节点之间添加边 |
| `amem info` |显示文件信息 |
| `amem traverse` | 从起始节点开始进行图遍历 |
| `amem search` |查找符合条件的节点 |
| `amem impact` | 运行因果影响分析|
| `amem resolve` |跟随 SUPERSEDES 链了解当前真相 |
| `amem export` |将图表导出为 JSON |
| `amem import` |从 JSON 导入 |
| `amem stats` |详细图表统计 |

**v0.2查询命令（新增9个）：**

|命令|描述 |
|:---|:---|
| `amem text-search` | BM25 文本搜索（1.58 ms @ 100K，带索引）|
| `amem hybrid-search` |通过 RRF 组合 BM25 + 矢量搜索 |
| `amem centrality` | PageRank、程度或介数中心性 |
| `amem path` |两个节点之间的最短路径（BFS 或 Dijkstra）|
| `amem revise` |反事实信念修正分析|
| `amem gaps` |检测推理弱点和差距|
| `amem analogy` |查找过去结构相似的模式 |
| `amem consolidate` |去重、矛盾链接、推理提升 |
| `amem drift` |追踪信念随时间的演变 |

所有命令都支持 `--json` 输出以供编程使用。

## 3. MCP 服务器（用于 Claude Desktop、VS Code、Cursor、Windsurf）

MCP 服务器将大脑作为 12 个工具、6 个资源和 4 个提示向任何 MCP 兼容的 LLM 客户端公开。

```bash
cargo install agentic-memory-mcp
```

### 配置克劳德桌面

添加到`~/Library/Application Support/Claude/claude_desktop_config.json`：

```json
{
  "mcpServers": {
    "agentic-memory": {
      "command": "agentic-memory-mcp",
      "args": ["serve"]
    }
  }
}
```

> 零配置：默认为 `~/.brain.amem`。使用 `"args": ["--memory", "/path/to/brain.amem", "serve"]` 覆盖。

### 配置 VS Code / 光标

添加到`.vscode/settings.json`：

```json
{
  "mcp.servers": {
    "agentic-memory": {
      "command": "agentic-memory-mcp",
      "args": ["serve"]
    }
  }
}
```

＃＃＃ 核实

连接后，法学硕士即可访问 `memory_add`、`memory_query`、`memory_traverse`、`memory_correct`、`memory_resolve`、`memory_similar` 等工具。通过要求 LLM 存储事实进行测试：

>“请记住，该项目使用 PostgreSQL 16。”

LLM 应调用 `memory_add` 并确认事件已存储。

有关完整的工具/资源/提示参考，请参阅 [MCP 服务器自述文件](crates/agentic-memory-mcp/README.md)。

## 4. Auto-installer（连接所有AI工具）

自动安装程序会扫描您的计算机中的 AI 工具，并将它们全部连接到共享的 AgenticMemory Brain 文件。

```bash
pip install amem-installer
```

＃＃＃ 用法

```bash
# Auto-detect and configure all tools
amem-install install --auto

# Preview what would be configured (dry run)
amem-install install --dry-run

# Check connection status
amem-install status

# Remove all configurations
amem-install uninstall

# Re-scan for new tools
amem-install update
```

### 它的作用

1. 扫描系统中已安装的 AI 工具
2. 在 `~/.brain.amem` 创建共享大脑文件
3. 配置每个工具以使用共享大脑（通过 MCP 服务器、配置文件或包装脚本）
4. 在进行更改之前备份所有修改的配置

所有修改都是附加的 - 现有配置永远不会被删除。

### 支持的工具

## 从源代码构建

```bash
git clone https://github.com/agentralabs/agentic-memory.git
cd agentic-memory

# Build entire workspace (core library + MCP server)
cargo build --release

# Install core CLI
cargo install --path crates/agentic-memory

# Install MCP server
cargo install --path crates/agentic-memory-mcp

# Install Python SDK (development mode)
cd python
pip install -e ".[dev]"

# Install auto-installer (development mode)
cd ../installer
pip install -e ".[dev]"
```

### 运行测试

```bash
# All workspace tests (core + MCP + bridge: 314 tests)
cargo test --workspace

# Core library only (179 tests)
cargo test -p agentic-memory

# MCP server only (119 tests)
cargo test -p agentic-memory-mcp

# Bridge integration tests (16 tests)
cargo test -p agentic-memory-bridge-tests

# Python SDK tests (104 tests)
cd python && pytest tests/ -v

# Installer tests (39 tests)
cd ../installer && pytest tests/ -v
```

## 故障排除

### `pip: command not found`

请改用 `pip3` 或 Python 的完整路径：

```bash
python3 -m pip install agentic-brain
```

### `amem: command not found` 在 `cargo install` 之后

确保 `~/.cargo/bin` 在您的路径中：

```bash
export PATH="$HOME/.cargo/bin:$PATH"
```

将此行添加到您的 `~/.zshrc` 或 `~/.bashrc` 中以使其永久化。

### `amem-install: command not found` 在 `pip install` 之后

该脚本可以安装到用户本地 bin 目录。尝试：

```bash
python3 -m amem_installer.cli install --auto
```

或者找到 pip 安装的位置：

```bash
python3 -m pip show amem-installer
```

### Python SDK 显示“未找到 amem 二进制文件”

首先安装 Rust 核心引擎：

```bash
cargo install agentic-memory-cli
```

或者如果您没有 Rust，则从源代码构建：

```bash
git clone https://github.com/agentralabs/agentic-memory.git
cd agentic-memory
cargo build --release
cp target/release/amem /usr/local/bin/
```
