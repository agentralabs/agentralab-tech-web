## 基准原则

- 在已知硬件上测量并记录硬件信息
- 冷启动与热启动分别测试
- 报告命令、数据集与配置
- 不只报延迟，也报告失败率

## 建议模板

<Command
  title="记录系统上下文"
  description="每份基准报告开头都应包含。"
  code={`uname -a
rustc --version
node --version
python --version`}
/>

<Command
  title="运行时冒烟基准"
  description="用 version/help 命令作为基础健康计时。"
  code={`time amem --version
time agentic-vision-mcp serve --help
time acb-mcp serve --help`}
/>

## 报告结构

| 字段 | 示例 |
|---|---|
| 运行时 | AgenticVision |
| 场景 | 高频域名 warm perceive |
| 主机 | Apple M 系列 / Linux x86_64 |
| 数据集/负载 | 25 URLs / 3 sessions |
| 延迟 p50/p95 | 180ms / 410ms |
| 失败率 | 0.8% |
| 备注 | 启用 warm cache |

<Callout type="success" title="可比性">
比较分支或版本时，请保持命令与负载规模一致；同时改变两者会让结果不可行动。
</Callout>
